# Product Boundary

Course2Knowledge Lite is a course-to-interactive-knowledge-base system.

It is designed to be complete enough to use and evaluate as a real product, but
it deliberately avoids closed-loop learning-coach behavior.

## Public Capabilities

- Bilibili course import.
- Transcript normalization.
- Lecture notes and knowledge card generation.
- Course library and lecture reader.
- Citation-based course Q&A.
- Course search.
- Notes, bookmarks, tags, and reading progress.
- Web Lite frontdesk.
- Feishu/Hermes Lite frontdesk.

## Removed Capabilities

- Automatic study-plan automation.
- Calendar projection.
- Reflective learning journals.
- Learner scoring analysis.
- Exercise-review workflow.
- Visual exercise interpretation workflow.
- Spaced-review state changes.
- Task-completion automation.
- Protected orchestration and role-governance package.
- Production evidence or personal study logs.

## Public Frontdesk Rule

The public frontdesks answer content questions and manage lightweight course
records. They must not pretend to know whether the learner has mastered the
material.

## Data Rule

The public store is local product data. It is not a production authority surface
and it does not inherit protected production state.
