真实案例链路

从一个合集 URL，到两个能用的学习入口。

这次案例围绕同一个 Bilibili 合集：导入边界、转写证据、课程库、Web 前台和 Hermes 前台能力都留下了截图证据。页面展示的是实际跑出来的链路，不是概念图。

  1. 01 导入合集

    把 Bilibili season URL 作为公开导入边界。

  2. 02 展开课程

    得到有序 lecture、标题、BV/source URL 和 import receipt。

  3. 03 沉淀证据

    把 timestamped transcript segments 作为可引用学习证据。

  4. 04 建立课程库

    持久化 course、lecture、segment、card、note、bookmark 和 progress。

  5. 05 Web 深读

    在 Web Lite 里阅读、搜索、问答、生成卡片和记录状态。

  6. 06 Hermes 前台

    通过公开 Hermes profile 验证同一工具边界可被对话前台调用。

One Store, Two Frontdesks

Web 负责看清楚，飞书/Hermes 负责随手问。

可检查的阅读工作台

Web Lite

  • 按 lecture 阅读转写证据。
  • 搜索和 Q&A 都显示 citation。
  • 笔记、书签、进度都落在同一个本地 course store。
可对话的学习入口

Feishu/Hermes Lite

  • 不复制一套聊天记忆，仍然读同一个课程库。
  • 工具层支持查课、搜索、问答、笔记、书签、进度和视觉证据发图。
  • 当前公开站只展示安全 smoke，不泄露生产聊天身份。

产品思想

课程不是缺内容，是内容被锁在时间轴里。

Course2Knowledge Lite 做的事，是把播放器里的时间线拆成可引用证据，把证据组织成课程知识库，再让学习者通过 Web 和 Feishu/Hermes 两个前台进入同一个知识空间。

Timeline / 时间线 Evidence / 证据 Knowledge Store / 知识库 Inspectable Web / 可检查阅读 Conversational Feishu / 可对话前台

公开边界

裁掉闭环，不是裁成玩具。

公开版保留课程知识化的主线，同时移除私有计划、反馈、练题和生产身份相关能力。

Lite 保留

  • Bilibili course import boundary
  • Transcript evidence and local course store
  • Lecture reader, search, citation Q&A
  • Knowledge cards from source segments
  • Notes, bookmarks, and reading progress
  • Web Lite and Feishu/Hermes Lite frontdesks

公开版移除

  • Private planning layer
  • Exercise feedback and visual exercise interpretation
  • Learner scoring and mastery mutation loops
  • Private orchestration and production identifiers
  • Production chat exports and personal study logs
  • Protected closed-loop automation
