Real case walkthrough

From one collection URL to two usable learning surfaces.

The public demo case uses a Bilibili season URL. The page keeps every step inspectable so a reviewer can see the import boundary, evidence boundary, store boundary, and frontdesk boundary instead of trusting a marketing summary.

  1. 01 Import collection

    Accept the public Bilibili season URL as the source boundary.

  2. 02 Expand lectures

    Resolve ordered lectures, titles, BV/source links, and import receipt.

  3. 03 Capture evidence

    Store timestamped transcript segments as citable learning evidence.

  4. 04 Build course store

    Persist course, lecture, segment, card, note, bookmark, and progress files.

  5. 05 Read on Web

    Project the store into reader, search, Q&A, cards, and learning state.

  6. 06 Ask from Feishu

    Use Hermes Lite to answer from the same public evidence boundary.

One store, two frontdesks

The Web surface is for deep inspection. Feishu is for learning in motion.

Inspectable workspace

Web Lite

  • Lecture reader with timestamped transcript evidence.
  • Search and Q&A with source citations visible.
  • Notes, bookmarks, and lightweight progress over the local store.
Conversational workspace

Feishu/Hermes Lite

  • Ask course questions inside the learner's chat environment.
  • Look up course status without opening the Web workspace.
  • Keep the same evidence boundary instead of a separate chat-only memory.

Product thought

The problem is not that courses lack content. The content is trapped in a timeline.

Course2Knowledge Lite treats a video course as raw material. It cuts the timeline into citable evidence, organizes that evidence into a local course knowledge store, and lets learners enter the same knowledge space through two different frontdesks: an inspectable Web workspace and a conversational Feishu workspace.

Timeline Evidence Knowledge Store Inspectable Web Conversational Feishu

Public boundary

Trimmed loop, not a thin demo.

The public version keeps the course-to-knowledge product line substantial while removing private closed-loop coaching machinery.

Included in Lite

  • Bilibili course import boundary
  • Transcript evidence and local course store
  • Lecture reader, search, citation Q&A
  • Knowledge cards from source segments
  • Notes, bookmarks, and reading progress
  • Web Lite and Feishu/Hermes Lite frontdesks

Removed from public release

  • Private planning layer
  • Exercise feedback and visual exercise interpretation
  • Learner scoring and mastery mutation loops
  • Private orchestration and production identifiers
  • Production chat exports and personal study logs
  • Protected closed-loop automation
